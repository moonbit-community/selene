## Description
Add a small internal performance baseline for the WebGPU 3D primitive renderer. Issue 35 reports poor browser performance for primitive-heavy scenes such as `gameparty/pacman3d`; before changing renderer structure, Selene should expose enough renderer-side counters to verify that later work reduces CPU submission pressure.

## Design
Add debug-only or internal runtime counters for 3D sections, draw commands, uploaded vertex bytes, and bind group creation hot spots. Keep this behind existing WebGPU runtime internals; do not add user-facing engine API unless needed for tests. Prefer deterministic white-box tests for command counting over timing-based tests.

## Acceptance Criteria
- [x] WebGPU 3D submission exposes or records per-frame counts for sections, 3D command groups, draw commands, uploaded vertex bytes, and render-path bind group creations.
- [x] A white-box test or equivalent deterministic check covers a repeated primitive scene and verifies the counters.
- [x] `moon test selene-webgpu/src --target js --deny-warn` passes.
- [x] `moon check --target all --deny-warn` passes or any unrelated failure is documented.
