## Description
The MoonBit-first WebGPU refactor needs explicit tests and audits so future changes do not move renderer policy back into embedded JavaScript.

## Design
Add white-box tests for MoonBit-owned resource keys, primitive mesh signatures, batching, retained policy, and lighting packet behavior. Add a lightweight source audit that reports large embedded JS blocks or high-level WebGPU policy terms inside JS glue.

## Acceptance Criteria
- [ ] Tests cover MoonBit-side resource key and batching behavior.
- [ ] Tests cover retained/static primitive telemetry after the migration.
- [ ] Tests cover WebGPU lighting packet behavior for full, cheap, and unlit modes.
- [ ] A source audit or documented check flags large embedded JS/policy regression in `selene-webgpu`.
- [ ] `moon check --target all --deny-warn` passes.
