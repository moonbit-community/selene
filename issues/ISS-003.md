## Description
Batch repeated untextured WebGPU 3D primitives with the same mesh/material pipeline state into instanced draws. Primitive-heavy scenes should not pay one draw command and one transformed vertex stream per cube, plane, sphere, or cylinder.

## Design
Add an internal instance submission path after ISS-002 provides reusable mesh buffers. Bucket render items by primitive mesh key, material color/emissive/alpha/unlit state, double-sided state, and shadow flags. Upload transforms and per-instance material data through an instance buffer, then draw one mesh with many instances. Preserve existing visual behavior and fallback to the current path when a primitive cannot be instanced.

## Acceptance Criteria
- [x] Repeated untextured primitives with compatible mesh and material state are submitted as instanced draws.
- [x] Per-instance transform, scale, color, alpha mode, emissive, unlit, and shadow receive/cast flags remain visually equivalent to the previous path.
- [x] Telemetry from ISS-001 shows fewer draw commands and less uploaded vertex data for a repeated primitive scene.
- [x] `moon test selene-webgpu/src --target js --deny-warn` passes.
- [x] `moon check --target all --deny-warn` passes or any unrelated failure is documented.
