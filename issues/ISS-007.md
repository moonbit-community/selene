## Description
Define the MoonBit-to-JavaScript boundary for `selene-webgpu` before moving implementation logic. The boundary should expose browser/WebGPU capabilities as direct syscalls or opaque handle operations, not high-level renderer behavior.

## Design
Introduce or outline a `webgpu_sys` style layer with functions for device/context setup, resource creation/release, buffer writes, texture/sampler/pipeline/bind-group creation, encoder/pass operations, draw calls, submit, and browser-only DOM/audio/input calls. Avoid functions that decide batching, mesh generation, lighting policy, or render ordering.

## Acceptance Criteria
- [ ] A concrete syscall boundary is represented in code or an issue-local implementation note with function groups and ownership rules.
- [ ] The boundary distinguishes opaque GPU handles from MoonBit-owned render/resource keys.
- [ ] Existing high-level JS entrypoints that should be retired are identified.
- [ ] No renderer policy is added to JavaScript while defining this boundary.
- [ ] `moon check selene-webgpu/src --target js --deny-warn` passes.
