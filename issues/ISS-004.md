## Description
Extend the WebGPU 3D batching path to textured primitives and repeated material bindings. The current textured path builds triangle meshes and creates bind groups in the render path, which is expensive for repeated textured objects.

## Design
Reuse the primitive GPU mesh cache from ISS-002 and the instance submission model from ISS-003. Add material/texture bucket keys that include base, emissive, metallic-roughness, occlusion, normal textures, sampler codes, double-sided state, and shadow state. Cache bind groups for stable material texture combinations and shadow texture state where safe.

## Acceptance Criteria
- [ ] Textured built-in primitives avoid per-item primitive mesh generation when their parameter set is cacheable.
- [ ] Compatible textured primitives are bucketed to reduce draw calls where possible.
- [ ] Stable material texture bind groups are reused instead of being recreated for every compatible command.
- [ ] Existing textured primitive, normal, tangent, alpha, and double-sided tests continue to pass.
- [ ] `moon test selene-webgpu/src --target js --deny-warn` passes.
