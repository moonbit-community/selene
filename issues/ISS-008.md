# ISS-008: Move WebGPU resource ownership to MoonBit

## Metadata
- Type: task
- Status: open

- Assignee: unassigned
- Created: 2026-06-01
- Updated: 2026-06-01

## Description
Move resource identity and lifecycle policy out of embedded JavaScript. MoonBit should decide GPU resource keys, reuse, invalidation, and release; JavaScript should only create, update, and destroy opaque GPU objects on request.

## Design
Create MoonBit-side records for buffer, texture, sampler, pipeline, bind group, and retained resource handles. JS stores actual browser objects behind opaque IDs. Resource cache keys and pruning rules live in MoonBit.

## Acceptance Criteria
- [ ] MoonBit owns resource cache keys and lifecycle decisions for at least buffers and primitive mesh buffers.
- [ ] JS exposes thin create/write/destroy operations using opaque handles.
- [ ] Existing primitive mesh cache behavior remains equivalent or better.
- [ ] Dynamic buffers still avoid destroying GPU buffers before submitted command encoders finish.
- [ ] `moon test selene-webgpu/src --target js --deny-warn` passes.
