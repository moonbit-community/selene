# ISS-002: Cache WebGPU primitive GPU meshes

## Metadata
- Type: task
- Status: closed

## Description
Stop regenerating and re-uploading identical built-in primitive mesh geometry for every WebGPU 3D primitive item. Cubes, planes, spheres, and cylinders should have reusable GPU mesh records keyed by primitive parameters and vertex layout.

## Design
Introduce an internal WebGPU primitive mesh cache. The cache should generate primitive vertex/index data once per stable parameter key, upload GPU buffers once, and reuse those buffers across frames. Keep the cache internal to the WebGPU backend. This issue is about mesh storage reuse only; full multi-instance drawing is handled by ISS-003.

## Acceptance Criteria
- [x] Built-in untextured cube, plane, sphere, and cylinder primitives can resolve reusable GPU mesh records and cached untransformed vertex templates before per-item submission.
- [x] Cache keys include geometry-affecting parameters such as size/radius/height/slices and vertex layout.
- [x] Cache invalidation is deterministic and does not leak old GPU buffers when the WebGPU runtime resets.
- [x] Existing 3D primitive rendering tests continue to pass.
- [x] `moon test selene-webgpu/src --target js --deny-warn` passes.
