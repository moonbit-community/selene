# ISS-011: Move WebGPU lighting packets and low-cost modes to MoonBit

## Description
WebGPU lighting policy should be decided in MoonBit. Embedded JavaScript should not choose full, cheap, or unlit behavior or decide light count limits beyond direct shader/uniform constraints.

## Design
Represent WebGPU lighting mode and filtered light packets in MoonBit. JS should receive final packed uniform data or opaque buffers. Keep mode behavior explicit and testable.

## Acceptance Criteria
- [ ] MoonBit owns `Full`, `Cheap`, and `Unlit` mode behavior.
- [ ] MoonBit decides directional, point, and spot light counts for each mode.
- [ ] JS only uploads final light uniform data or binds handles.
- [ ] Behavior is covered by white-box tests for light packet counts.
- [ ] `moon test selene-webgpu/src --target js --deny-warn` passes.
