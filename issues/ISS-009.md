# ISS-009: Move primitive mesh generation and cache keys to MoonBit

## Metadata
- Type: task
- Status: open
- Priority: 1
- Labels: webgpu, render3d, mesh
- Assignee: unassigned
- Created: 2026-06-01
- Updated: 2026-06-01
- External ref: none

## Description
Primitive mesh construction for WebGPU should be MoonBit-owned. Embedded JavaScript should not generate cube, sphere, cylinder, or plane vertices or decide primitive cache keys.

## Design
Reuse or consolidate existing MoonBit primitive mesh builders and serialize/upload their vertex data through the syscall layer. Keep cache keys and geometry signatures in MoonBit. JS should only receive typed vertex buffers or buffer handles.

## Acceptance Criteria
- [ ] Cube, sphere, cylinder, and plane WebGPU primitive geometry is generated or selected in MoonBit.
- [ ] Primitive cache keys are built in MoonBit.
- [ ] WebGPU and raylib primitive shape semantics remain aligned.
- [ ] Existing primitive winding/height/scale regression tests remain green.
- [ ] `moon test selene-webgpu/src --target js --deny-warn` passes.
