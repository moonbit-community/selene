## Description
After resource, primitive, batching, and lighting ownership moves to MoonBit, remove the remaining high-level renderer logic from `canvas.mbt`. The file should become a thin WebGPU and DOM syscall implementation rather than a renderer runtime.

## Design
Delete or shrink embedded JS functions that contain renderer policy. Keep only direct browser/WebGPU operations and minimal object-handle plumbing. Split MoonBit facade files if needed, but do not move policy into standalone JavaScript files.

## Acceptance Criteria
- [ ] `canvas.mbt` no longer contains primitive mesh builders, batch compatibility decisions, retained policy, or lighting mode policy.
- [ ] `canvas.mbt` is substantially smaller and organized as syscall glue.
- [ ] Remaining embedded JS snippets are short direct calls or object-handle operations.
- [ ] Existing WebGPU 2D/3D/audio/input behavior still works.
- [ ] `moon check --target all --deny-warn` passes.
