# ISS-010: Move WebGPU 3D batching and retained policy to MoonBit

## Description
Batch formation, instancing buckets, textured command compaction, and retained static primitive policy are renderer decisions and should live in MoonBit, not embedded JavaScript.

## Design
Build a MoonBit render command IR for WebGPU 3D submissions. Convert `RenderFrame3D` into explicit batches before crossing the JS boundary. JS receives already-decided draw batches and only binds handles, sets buffers, and issues draw calls.

## Acceptance Criteria
- [x] MoonBit forms untextured instanced primitive batches.
- [x] MoonBit forms textured primitive batches or explicitly marks paths that cannot batch yet.
- [x] MoonBit owns retained/static instance signatures and pruning policy.
- [x] JavaScript no longer decides compatible primitive batches.
- [x] Telemetry continues to show reduced draw/upload pressure for repeated primitive-heavy scenes.
- [x] `moon test selene-webgpu/src --target js --deny-warn` passes.
